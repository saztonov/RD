# Git Push с автоматическим сообщением на русском

Выполни следующие шаги для создания коммита и пуша:

## 1. Анализ изменений

Выполни параллельно:
- `git status` — просмотр изменённых файлов
- `git diff` — unstaged изменения
- `git diff --staged` — staged изменения
- `git log --oneline -5` — последние коммиты для понимания стиля

## 2. Добавление файлов

Добавь изменённые файлы в staging. Предпочитай добавлять конкретные файлы вместо `git add -A`.

**Не добавляй:**
- `.env` и файлы с секретами
- Большие бинарные файлы
- Временные файлы

## 3. Генерация сообщения коммита

На основе анализа изменений сгенерируй сообщение на **русском языке** в формате conventional commits:

| Префикс | Использование |
|---------|---------------|
| `feat:` | новая функция |
| `fix:` | исправление ошибки |
| `docs:` | документация |
| `refactor:` | рефакторинг без изменения функционала |
| `perf:` | оптимизация производительности |
| `chore:` | технические задачи, зависимости |
| `test:` | тесты |

**Примеры:**
- `feat: добавить поддержку экспорта в PDF`
- `fix: исправить ошибку валидации форм`
- `refactor: упростить логику обработки блоков`

## 4. Создание коммита

Создай коммит с HEREDOC для правильного форматирования:

```bash
git commit -m "$(cat <<'EOF'
<сообщение>

Co-Authored-By: Claude <noreply@anthropic.com>
EOF
)"
```

## 5. Push

Выполни `git push` для отправки изменений на GitHub.

Если ветка не отслеживается, используй:
```bash
git push -u origin <имя-ветки>
```
